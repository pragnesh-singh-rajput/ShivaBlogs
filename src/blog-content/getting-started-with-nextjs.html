Next.js 14 continues to build upon the foundations laid by its predecessors, offering a powerful and flexible framework for building modern web applications. This guide will walk you through setting up a new Next.js 14 project, dive deep into the App Router, and explore its core concepts such as Server Components, layouts, pages, data fetching, and more.

Prerequisites

Before we begin, ensure you have the following installed:

  • Node.js (version 18.17 or later recommended). You can download it from nodejs.org.
  • A basic understanding of React and JavaScript.

Project Setup

Creating a new Next.js application is straightforward using the official command-line tool. Open your terminal and run:

npx create-next-app@latest my-next-app

You'll be prompted with a few questions, such as whether to use TypeScript, ESLint, Tailwind CSS, etc. For this guide, we'll assume you've chosen the defaults, including TypeScript and the App Router.

Once the installation is complete, navigate to your project directory:

cd my-next-app

And start the development server:

npm run dev

Open your browser to http://localhost:3000 to see your new Next.js app in action!

Understanding the App Router

The App Router, introduced in Next.js 13 and refined in version 14, is a paradigm shift from the older Pages Router. It leverages React Server Components by default and offers a more intuitive way to structure your application through file-system-based routing within the app directory.

Key benefits include improved performance by reducing client-side JavaScript, better data fetching capabilities, and enhanced support for layouts and streaming.

Explore more about the fundamentals in the Next.js Routing Documentation.

Core Concepts of the App Router

1. Server Components by Default

Components within the app directory are React Server Components (RSCs) by default. This means they run on the server, allowing them to directly access backend resources (like databases) and render HTML that is sent to the client. This reduces the amount of JavaScript bundled and sent to the browser, leading to faster initial page loads. Read more on Server Components.

2. Client Components

For interactivity, event listeners (like onClick), or browser-only APIs, you'll need Client Components. You can opt-in to client-side rendering by adding the "use client" directive at the top of your component file. Client Components are rendered on the server for the initial load (SSR/SSG) and then "hydrated" on the client to become interactive. Learn when to use them in the Client Components documentation.

3. Pages (page.tsx)

A page.tsx file defines the unique UI for a specific route segment. For example, app/dashboard/page.tsx maps to the /dashboard URL. Each page is a Server Component by default. See Pages documentation.

4. Layouts (layout.tsx)

A layout.tsx file defines a shared UI that wraps multiple pages or segments. Layouts preserve state, remain interactive, and do not re-render when navigating between child segments. For example, app/layout.tsx is the root layout for your entire application. You can nest layouts, e.g., app/dashboard/layout.tsx. Details can be found in the Layouts documentation.

5. File-System Routing Conventions

The App Router uses a set of special file names to create UI with specific behavior:

  • page.tsx: Creates a publicly accessible URL path.
  • layout.tsx: Defines shared UI for a segment and its children.
  • loading.tsx: Creates loading UI for a segment using React Suspense. See Loading UI documentation.
  • error.tsx: Defines error UI for a segment using React Error Boundaries. See Error Handling documentation.
  • not-found.tsx: Defines UI for when the notFound() function is thrown within a segment. See notFound UI documentation.
  • route.ts: For creating API endpoints (similar to API routes in the Pages Router). See Route Handlers documentation.

6. Dynamic Segments

You can create dynamic routes by naming a folder with square brackets, e.g., app/blog/[slug]/page.tsx. The slug parameter will be available in your page component's props. Learn more about Dynamic Routes.

7. Route Groups

Route groups allow you to organize your routes without affecting the URL path. Create a folder with parentheses, e.g., app/(marketing)/about/page.tsx will still map to /about. This is useful for creating different layouts for sections of your site. See Route Groups documentation.

Data Fetching

With Server Components, data fetching becomes more integrated. You can use async/await directly within your Server Components.

async function getData() {
  const res = await fetch('https://api.example.com/...');
  if (!res.ok) {
    throw new Error('Failed to fetch data');
  }
  return res.json();
}

export default async function Page() {
  const data = await getData();
  // ... render data
}

Next.js extends the native fetch API to provide automatic caching and revalidation options. This simplifies data fetching patterns and allows fine-grained control over how your data is handled. Consult the Data Fetching documentation for detailed patterns.

Server Actions

Server Actions allow you to run server-side code directly from Client Components, often triggered by form submissions or button clicks, without needing to manually create API endpoints. Define an asynchronous function with the "use server" directive.

'use server';

export async function submitForm(formData: FormData) {
  // ... process form data, interact with database
  const email = formData.get('email');
  // ...
}

Server Actions simplify mutations and form handling. Explore them further in the Server Actions documentation.

Styling Your Application

Next.js supports various styling methods:

  • Global CSS: Import stylesheets into your root layout (app/layout.tsx).
  • CSS Modules: Scope styles locally to components (*.module.css).
  • Tailwind CSS: A utility-first CSS framework that integrates seamlessly.
  • CSS-in-JS Libraries: Many popular libraries are compatible.

Choose the method that best suits your project's needs. More information is available on the Styling documentation page.

Conclusion

Next.js 14, with its App Router and focus on Server Components, provides a robust and efficient platform for modern web development. This guide has touched upon the foundational aspects to get you started. The official Next.js documentation is an invaluable resource for diving deeper into specific features and advanced patterns.
